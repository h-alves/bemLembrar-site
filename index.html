Bem Lembrar

Um app que facilita a gestão de seus clientes e suas preferências, permitindo o foco no relacionamento com seus clientes.

O Bem Lembrar atua em cima da descentralização das informações de contato, que geram um desafio na manutenção do relacionamento cliente-consultora.

Assim, trazendo uma solução com diversas features para facilitar o dia-a-dia das consultoras de beleza. Como por exemplo:

  • Registrar as preferências de cada cliente
  • Notificações em datas importantes
  • Mensagens personalizadas para cada cliente.

Focando em uma interface simples de ser navegada e estratégias para agilizar o registro de clientes e suas preferências, o Bem Lembrar vem como um respiro para um mercado cheio de ferramentas complexas.

Além disso, o app comporta o Dynamic Type desde o xSmall até o xxxLarge, sem comprometer nenhuma informação.

User Experience

Fluxo do usuário

O primeiro passo do processo de design foi realizar o fluxo principal do usuário, mapeando a sequência ideal de telas.

Também nesse processo foi realizado um extenso onboarding que iria abordar o funcionamento de diversas features do app usando um sistema parecido com o de um chat de aplicativos de mensagem, a fim de manter um senso de familiaridade para as consultoras.

Wireframes

Então com o fluxo definido, foram realizados diversos wireframes para as telas essenciais para o app, o que agilizou muito o código, possibilitando a publicação do app no Test Flight para o MVP.

Teste de Usabilidade

Depois de definido o fluxo e as telas, e ainda durante o processo de criação das telas finais, foi realizado um teste de usabilidade, o que ajudou a entender quais partes do app deveriam continuar e também algumas melhorias interessantes para o app.

Para realizar esse teste de usabilidade, nós desenvolvemos um roteiro que buscasse abordar todas nossas dúvidas em relação a cada parte do Bem Lembrar, tanto o método adotado para registrar preferências dos clientes, a aparência desejada do app e um teste de funcionalidade.

Fluxo revisado

Após muitas etapas, o fluxo foi revisado trazendo um onboarding mais sucinto e aplicando os momentos ideias de se pedir as permissões necessárias para o app funcionar.

User Interface

Benchmarking

Já no processo de criação da identidade visual, foram analisadas diversas referências, principalmente apps que gerassem em torno do mesmo tema, e até mesmo apps que as consultoras já estavam acostumadas a usar.

IDV

Inicialmente, após a análise de diversos similares, começamos o processo de desenvolver nossa própria identidade visual, buscando trazer familiaridade com elementos visuais à área da beleza.

Assim, foi escolhida a paleta de cores e outros elementos visuais, escolhemos o rosa e o verde, sendo cores complementares e que trazem com um tom de delicadeza a interface.

Naming

Quanto ao naming, surgiu o nome Bem Lembrar, buscando associar à expressão brasileira bem lembrado a principal função do nosso app, manter as informações das suas clientes sempre frescas na sua mente.

Ilustrações

Em busca de trazer um tom mais agradável foram utilizadas flores em todas as partes do app, além disso, foram criadas inúmeras ilustrações contendo esses mesmo elementos visuais.

Telas

A partir dos wireframes realizados, foram feitas as telas em alta fidelidade utilizando auto-layout, já buscando facilitar o processo de criar as telas no código. Além disso, o app todo se conforma tanto ao Light quanto ao Dark Mode, para atender as preferências dos usuários.

Dynamic Type

Durante a produção das telas o que mais exigiu atenção foi adequar o layout para Dynamic Type sem comprometer as informações e a experiência, visando sempre manter a acessibilidade.

Código

A parte de desenvolvimento envolveu diversas tecnologias distintas que juntas trouxeram as múltiplas features do Bem Lembrar.

De início foi utilizada o framework Contacts da apple o que permitiu a coleta da lista de contatos do celular do usuário, assim facilitando o registro de cada cliente. Afinal, os dados essenciais deles já vinham direto dos contatos.

Após registrar os clientes e preferências é possível realizar dois modos de contato com cada cliente. Ligações, que foi possível graças a chamada de uma URL de ligação com o número da pessoa.

Também, é possível enviar mensagens personalizadas com o UIActivityViewController, o que traz a tela de share do próprio iPhone com a mensagem customizada para ser enviada para seus contatos.

func callClient() {
    let telephone = "tel://"
    let formattedString = telephone + client.number
    guard let url = URL(string: formattedString) else { return }
    UIApplication.shared.open(url)
    
    registerContact()
}
    
func shareMessage() {
    self.shareText = "ooi querida \(client.fullName)🫶\ncomo você está? Está precisando de algum dos nossos produtos??🥰💐"
    if client.birthday?.getMonthDay() == Date().getMonthDay() {
        self.shareText = "Ooi \(client.fullName)! 🥳✨\nFeliz aniversário, te desejo muito amor, paz e sabedoria nessa nova
				primavera da sua vida! 🥰 💐 Obrigada pela confiança na minha consultoria e que sua luz nunca se apague"
    }
    
    let activityVC = UIActivityViewController(activityItems: [shareText], applicationActivities: nil)
    UIApplication.shared.currentUIWindow()?.rootViewController?.present(activityVC, animated: true, completion: nil)
    
    registerContact()
}

Além disso, foi implementado um manager para armazenar as funções de notificação do app, podendo gerar diferentes tipos de notificação, como as UNCalendarNotificationTrigger e as UNTimeIntervalNotificationTrigger, e com diversos textos distintos a fim de passar as mensagens necessárias.

Por fim, tanto a lista de clientes, com seus dados e preferências, quanto as informações do usuário são salvas no celular da pessoa usando UserDefaults, o que se provou um desafio no código, principalmente por conta do mal uso de singletons, gerando a necessidade de usar Combine para garantir as informações corretas na tela.
